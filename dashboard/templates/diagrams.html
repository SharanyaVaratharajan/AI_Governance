Governance flow, safe preflight controls, and evidence lifecycle.

Safe governance gateway

Inspect before a model call

flowchart TD
  A[User input] --> B[Gateway API /governance/run]
  B --> C[PII detection and sanitization]
  C --> D[Phishing preflight]
  D -->|ALLOW| E[Model call]
  D -->|REDACT| E
  D -->|REVIEW or BLOCK| F[Skip model call]
  E --> G[Risk engine]
  F --> G
  G --> H[Sanitized run record]
  H --> I{Review required?}
  I -->|Yes| J[Open incident]
  I -->|No| K[Cleared result]

System-level PII policy determines whether recognised PII is redacted, blocked, or allowed in approved workflows.

Evidence and review lifecycle

From run record to governance reporting

flowchart TD
  A[Sanitized model run] --> B[Safety flags]
  A --> C[Preflight decision]
  B --> D[Audit event]
  C --> D
  D --> E[Audit log]
  A --> F[Runs dashboard]
  F --> G[Run detail: safe preview]
  G --> H[Incident review]
  H --> D
  F --> I[Filtered CSV export]
  H --> J[Incidents dashboard]
  J --> K[Filtered CSV export]

Original sensitive values are withheld; reviewers see the sanitized payload and typed redaction evidence.
